步骤三：配置编辑器核心设置

3.3.1 打开设置面板

按 Ctrl + ,（逗号）打开设置面板。VS Code 的设置分为两个级别：

级别 说明 适用场景
用户设置 对所有项目生效 个人偏好配置，如主题、字体、自动保存等
工作区设置 仅对当前项目生效 项目特定配置，如项目专属的格式化规则、Python 解释器路径等

本次实验配置用户级别设置，以便在所有项目中复用。

3.3.2 配置自动保存

在设置搜索框中输入 files.autoSave，在下拉选项中选择 **afterDelay**。

功能说明：开启自动保存后，VS Code 会在用户停止编辑的短暂延迟（默认 1000 毫秒）后自动保存文件。这一功能具有以下优势：

  • 避免内容丢失：防止因忘记手动保存（Ctrl+S）而导致的代码或文档内容丢失
  • 提升工作效率：减少手动保存的操作次数，专注于内容创作
  • 配合实时预览：自动保存配合 Markdown Preview Enhanced 的自动刷新，实现真正的“所见即所得”

其他可选值：

  • off：关闭自动保存（需要手动 Ctrl+S 保存）
  • onFocusChange：编辑器失去焦点时自动保存
  • onWindowChange：窗口失去焦点时自动保存

3.3.3 配置自动换行

在设置搜索框中输入 editor.wordWrap，在下拉选项中选择 **on**。

功能说明：开启自动换行后，长文本会在编辑器视口边缘自动换行显示，无需水平滚动条即可阅读完整内容。

适用场景：

  • Markdown 文档写作（段落通常较长）
  • README 文件编辑
  • 代码注释阅读
  • 日志文件查看

其他可选值：

  • off：关闭自动换行（需要水平滚动查看长行）
  • on：根据视口宽度自动换行
  • wordWrapColumn：在指定列数处换行
  • bounded：在视口宽度和指定列数中较小值处换行

3.5.1 Git 仓库的概念

Git 仓库（Repository）是 Git 版本控制系统的核心概念，它是一个被 Git 管理的项目文件夹，其中包含了项目的所有文件以及它们完整的修改历史记录。

Git 仓库的主要组成部分：

组成部分 说明
.git 隐藏文件夹 存储所有版本信息的核心目录，包含 commit 对象、分支引用、配置等，任何情况下都不要手动修改或删除
工作区（Working Directory） 当前正在编辑和查看的项目文件
暂存区（Staging Area / Index） 准备提交的文件列表
本地仓库（Local Repository） 存储所有历史提交记录

问题3：无法修改root所属文件权限/所有者

  • 原因：仅文件所有者或root管理员有权限变更文件归属与权限

6、出现问题

  1. bash deploy.sh 命令报错：
    • 现象：Windows 终端提示 'bash' 不是内部或外部命令。
    • 原因：Windows 的 CMD 终端不识别 Linux 的 bash 语法。
    • 解决：利用 Windows 自带的 scp 命令替代原方案，手动完成传输。
  2. git push 初期连接失败：
    • 现象：终端提示 Failed to connect to github.com port 443。
    • 原因：国内网络环境访问 GitHub 经常不稳定。
    • 解决：更换网络环境后重试，成功完成提交。
  3. Nginx 配置文件冲突警告：
    • 现象：执行 nginx -t 时提示 conflicting server name "_"。
    • 解决：该警告为默认冲突提示，不影响站点实际运行，忽略即可。
