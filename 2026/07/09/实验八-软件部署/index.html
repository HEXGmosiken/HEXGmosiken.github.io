    实验八 软件部署

    Posted on 2026-07-09

    报告人： 陈玉婷 25051506

    实验八：Web站点部署

    1、实验目的

    本次实验旨在综合运用前七个实验所学的各类工具，完整体验和实现 Web 静态站点（Hexo 博客）从本地开发到云端部署的全过程。通过本次实验，掌握 Git 版本控制、自动化部署脚本编写、GitHub Actions 持续集成配置，以及 Linux 虚拟机下 Nginx 服务器的安装与站点托管。

    2、实验环境

    • 操作系统：Windows 11 专业版
    • 开发工具：Visual Studio Code, Node.js v20, Hexo
    • 版本控制：Git, GitHub
    • 运行环境：Oracle VirtualBox, Ubuntu Server 22.04 LTS
    • 网络环境：NAT + Host-Only 双网卡模式（Linux 虚拟机网段 192.168.56.0/24）

    5、知识总结

    核心环节 使用工具/技术 主要作用
    代码托收 Git + GitHub 实现代码的版本管理与远程备份
    自动化构建 GitHub Actions + deploy.yml 代码推送后自动执行 Hexo 生成与部署
    脚本化部署 deploy.sh 本地一键完成清理、打包与推送流程
    服务端托管 Nginx 提供 Web 服务器能力，托管静态 HTML 页面
    跨平台传输 scp 将 Windows 本地的文件安全传输至 Linux 服务器

    6、出现问题

    1. bash deploy.sh 命令报错：
      • 现象：Windows 终端提示 'bash' 不是内部或外部命令。
      • 原因：Windows 的 CMD 终端不识别 Linux 的 bash 语法。
      • 解决：利用 Windows 自带的 scp 命令替代原方案，手动完成传输。
    2. git push 初期连接失败：
      • 现象：终端提示 Failed to connect to github.com port 443。
      • 原因：国内网络环境访问 GitHub 经常不稳定。
      • 解决：更换网络环境后重试，成功完成提交。
    3. Nginx 配置文件冲突警告：
      • 现象：执行 nginx -t 时提示 conflicting server name "_"。
      • 解决：该警告为默认冲突提示，不影响站点实际运行，忽略即可。
