    5、知识总结

    核心命令/概念 作用/含义 对应考点
    sudo adduser 用户名 交互式创建新用户，并自动生成 /home/用户名 目录 账户配置
    sudo usermod -aG sudo 用户名 将用户添加到 sudo 组，使其拥有管理员权限 账户配置
    su - 用户名 切换用户身份，并加载该用户的 Home 环境 Home 配置
    /home/用户名 用户专属的家目录，是权限隔离的基础空间 Home 配置
    chmod 755 文件名 数字法修改权限：7(rwx) 5(r-x) 5(r-x) 权限配置
    ls -l 查看文件的详细权限（rwx）、所有者及所属组 权限配置
    export PATH=$PATH:新路径 临时将新目录添加到 PATH，让该目录下的程序可直接运行 环境变量
    ~/.bashrc 用户级 Shell 配置文件，修改后执行 source 可永久保存配置 环境变量

    6、出现问题

    问题1：普通用户执行系统操作提示 Permission denied

    • 现象：普通用户执行系统相关操作，命令报错无执行结果
    • 原因：普通用户不具备修改系统配置的权限
    • 解决：执行管理类命令前添加 sudo 提权；或使用 sudo -i 切换至 root 用户操作

    问题2：export 设置PATH重启终端失效

    • 现象：临时配置PATH后关闭终端，自定义脚本无法直接调用
    • 原因：export 仅临时生效，终端关闭配置自动丢失
    • 解决：将 export PATH=$PATH:新路径 写入 ~/.bashrc，执行 source ~/.bashrc 永久生效

    问题3：无法修改root所属文件权限/所有者

    • 现象：执行 chmod、chown 修改系统文件失败
    • 原因：仅文件所有者或root管理员有权限变更文件归属与权限
    • 解决：操作命令前增加 sudo 提升管理员权限
