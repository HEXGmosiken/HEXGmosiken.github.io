    实验四 Web站点构建

    Posted on 2026-07-09

    报告人： 张婷玥 25051502

    1、实验目的

    本次实验在实验二搭建的Hexo环境基础上，完成一个完整静态博客站点的构建与部署。具体包括：初始化博客项目、编写Markdown文章、配置主题、本地预览，并通过GitHub Actions实现自动化部署到GitHub Pages，从而理解静态站点生成器（SSG）的工作流程及持续集成/部署（CI/CD）的基本思想。

    2、实验环境

    项目 版本/配置
    操作系统 Windows 11
    Node.js v24.18.0
    npm 11.16.0
    Git 2.51.0
    Hexo CLI 4.3.2
    文本编辑器 Visual Studio Code 1.85
    网络环境 校园网

    3、实验过程

    步骤一：环境验证

    步骤二：安装Hexo CLI

    步骤三：初始化博客项目

    在D:\Blog（或你选择的目录）执行 hexo init my-blog，进入目录执行 npm install。

    步骤四：本地预览

    hexo server，访问 http://localhost:4000 查看默认首页。

    步骤五：创建并编写博客文章

    hexo new “我的第一篇博客”，用VS Code编辑，添加内容。

    步骤六：更换主题（以Next为例）

    git clone https://github.com/theme-next/hexo-theme-next themes/next

    修改 _config.yml 中 theme: next

    执行 hexo clean && hexo generate && hexo server 预览新主题。

    步骤一到六均已在实验二中完成

    步骤七：部署到GitHub Pages（自动化）

    生成Personal Access Token，并添加到仓库Secrets中。

    推送源码，触发Actions自动构建并部署。

    访问 https://github.com/HEXGmosiken/HEXGmosiken.github.io 查看上线博客。

    四、实验结果

    五、知识总结

    5.1 静态站点 vs 动态站点

    对比项 静态站点（SSG） 动态站点
    页面生成时机 构建时预生成 用户访问时实时生成
    数据库 不需要 需要
    访问速度 快 慢
    安全性 高 需额外防护

    5.2 Hexo 核心目录

    目录/文件 作用
    _config.yml 站点全局配置（标题、URL、主题、部署）
    source/_posts/ 存放 Markdown 文章
    themes/ 存放主题（如 Next）
    public/ 生成的静态文件（部署内容）

    5.3 常用命令

    命令 简写 作用
    hexo new "标题" hexo n 新建文章
    hexo generate hexo g 生成静态文件
    hexo server hexo s 本地预览
    hexo deploy hexo d 部署到远程

    5.4 CI/CD 在本实验中的体现

    • CI（持续集成）：推送源码后，GitHub Actions 自动执行 hexo generate 构建。
    • CD（持续部署）：构建完成后自动将 public/ 推送到 用户名.github.io 仓库，完成上线。

    六、出现问题

    GitHub Actions构建失败，提示Token权限不足


    重新生成Token，勾选所有 repo 权限

    部署后访问 用户名.github.io 显示404。

    重命名仓库，检查部署插件配置后发现代码中仓库名写错了。

    七、心得体会

    技术层面：

    完整走通了“环境准备→项目初始化→内容编写→主题配置→本地预览→部署上线”的Web应用开发全流程，对SSG有了切身体会。

    掌握了GitHub Actions自动化部署，理解了CI/CD如何提升开发效率（修改代码后自动构建发布，无需手动上传）。

    认识到配置文件的统一管理（_config.yml）和版本控制（Git）在实际项目中的重要性。

    思政层面：

    GitHub Pages为个人开发者提供了免费的静态托管服务，降低了技术分享的门槛，这体现了互联网的开放精神。

    国内类似的平台（如Gitee Pages、CODING Pages）也在发展，但功能丰富度和生态尚需完善。通过对比，认识到国产软件在替代化进程中仍有很长的路要走，作为计算机专业学生，应关注国产基础软件（如代码托管平台、CI/CD工具）的发展，并积极参与贡献。

    开源协议（如MIT）保障了代码的自由使用和修改，我们既要享受开源红利，也要遵守规则，回馈社区。
